     ```yaml
       sum by (name) (rate(container_cpu_usage_seconds_total{name=~".*"}[5m])) * 100 > 400
     ```

  6. **DS01ContainerHighMemory** — Uses `ds01_container_memory_percent` which does NOT exist.

     ```yaml
     expr: |
       (container_memory_usage_bytes{name=~".+"} / container_spec_memory_limit_bytes{name=~".+"}) > 0.9
     ```

     Filter out containers without memory limits (spec = 0) by adding `container_spec_memory_limit_bytes > 0` in the expression.

  Audit the recording rules file. The recording rules are mostly well-structured (built during Phase 1), but need validation:

  1. **Verify GPU-hours attribution rules are correct:** The `ds01_cost_attribution` group contains `ds01:user_gpu_seconds` which computes `sum by (user) (ds01_gpu_allocated) * 60`. This is correct — it samples every minute and each allocated GPU adds 60 seconds. Dashboard query `increase(ds01:user_gpu_seconds[1h]) / 3600` gives GPU-hours per user.

  2. **Verify MIG mapping rules:** The `ds01_mig_mapping` group joins DCGM profiling metrics with DS01 slot info. These are correct and tested in Phase 1. No changes needed.

  3. **Add missing HELP comments** where absent — recording rules don't have native HELP but the comment structure helps documentation.

  4. **Add a new recording rule for lifecycle events** (bridges to the new exporter metrics):
     ```yaml
     # Lifecycle enforcement activity
     - name: ds01_lifecycle_metrics
       interval: 1m
       rules:
         # Total enforcement events in last 24h (for dashboard stat panel)
         - record: ds01:lifecycle_events_24h
           expr: sum(ds01_lifecycle_events_total) or vector(0)

         # Enforcement events by action (for trend panels)
         - record: ds01:lifecycle_kills_24h
           expr: |
             sum(ds01_lifecycle_events_total{action=~"idle_kill|runtime_kill|oom_kill"}) or vector(0)
     ```

  5. **Validate the `or vector(0)` fallback pattern:** Several rules use `or vector(0)` which produces 0 instead of "no data" when the upstream metric is missing. This is correct for counters and totals but misleading for utilisation percentages. Review:

     - Other `or vector(0)` usages for counts are fine (0 is correct when nothing exists).

  6. No structural changes to existing recording rules — they are architecturally sound. Only add the new lifecycle recording rule group and improve comments.
